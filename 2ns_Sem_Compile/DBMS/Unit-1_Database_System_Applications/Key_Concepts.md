# Key Concepts

## Database

A database is an organized collection of related data.

## DBMS

DBMS is the software used to create, store, update, and retrieve data from a database.

## Schema and Instance

- **Schema**: Overall design of the database
- **Instance**: Data stored in the database at a particular time

## Metadata

Metadata is data about data, such as table names, column names, and data types.

## DBA

The **Database Administrator** manages:

- Security
- Backup and recovery
- User authorization
- Performance tuning

## Query Language

- **DDL**: `CREATE`, `ALTER`, `DROP`
- **DML**: `INSERT`, `UPDATE`, `DELETE`
- **DQL**: `SELECT`
- **DCL**: `GRANT`, `REVOKE`
- **TCL**: `COMMIT`, `ROLLBACK`, `SAVEPOINT`

## Importance of Database Design

- Reduces redundancy
- Improves consistency
- Supports efficient retrieval
- Helps maintain integrity

## Example

```text
Good design -> less duplication -> fewer anomalies -> easier maintenance
```

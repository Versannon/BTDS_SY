# Unit 2: Relational Model

## Definition

The **Relational Model** represents data in the form of relations, which are commonly shown as tables.

## Basic Terminology

| Term | Meaning |
|---|---|
| Relation | Table |
| Tuple | Row |
| Attribute | Column |
| Domain | Set of allowed values |
| Degree | Number of attributes |
| Cardinality | Number of tuples |

## Integrity Constraints

Integrity constraints maintain correctness and consistency of data.

### Types

- Domain constraint
- Key constraint
- Entity integrity
- Referential integrity
- Check constraint
- Unique constraint
- Not null constraint

## Keys in Relational Model

- Super key
- Candidate key
- Primary key
- Alternate key
- Composite key
- Foreign key

## Example Relation

```text
EMPLOYEE(EmployeeID, EmployeeName, DepartmentID, Salary)
```

## Relational Query Languages

### Procedural

- Relational Algebra
- Tells what operation to perform and how to combine them

### Non-Procedural

- Relational Calculus
- Tells what result is needed, not the procedure

## Exam Notes

- Define relation, tuple, and attribute clearly.
- Write at least 4 integrity constraints with examples.
- Mention E. F. Codd when introducing the relational model.
